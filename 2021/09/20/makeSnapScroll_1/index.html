Snapscroll 애니메이션 추가하기 -1

Created2021-09-20|Updated2021-09-20|Blog
|Word count:413|Reading time:2min|Post View:

사전 조사

hexo blog의 경우 markdown을 html로 변환하기 때문에 포스트의 내용이 page-content란 이름으로 div 태그 아래에 저장되고 뿌려진다.

즉 내 블로그 같은 경우 페이지 단위가 아니라 소제목 단위로 애니메이션을 발동 시키면 비슷하게 구현할 수 있을거라 생각한다

주로 스냅 스크롤은 fullPage.js나 이번에 CSS 업데이트하면서 추가된 CSS-Snap-Scroll 기능을 이용한다.

안되면.. 자바스크립트라던가.

javascript로 도전!

뷰포트 기준으로 태그들을 정렬하면 되지 않을까? - 실패

아이디어는 다운스크롤 할 때 현재 View port 내 h2 태그(마크다운에서 ##은 h2 tag로 렌더링 됨)가 있다면 이동하는 것이다

  1. 현재 Viewport 내의 h2 tag를 검색
  2. top이 0에 가장 가까운 tag 검색(0은 제외)

실패 원인

h2 tag의 top이 최소인 걸 골라 이동했더니, 최소였던 값의 위치가 바뀌고 새로운 최솟값이 생기면 또 거기로 이동하고 무한 반복을 해버린다.

function 설명

downScroll()

스크롤 위치 값을 비교해서 다운 스크롤 할 때만 true를 반환하는 함수이다

findMinTag()

findMinTag 함수는 0 이 아닌 최솟값을 골라주는 함수이다
(top이 0 이면 뷰포트의 최상단에 위치하기 때문이다. 난 태그를 top = 0 에 위치시키고 싶다)

처음 생각엔 h2 태그들을 노드리스트로 받아오고, 노드 안에는 해당 요소의 여러 정보들이 들어있으니 top 추출하는 과정에서 바로 최솟 값을 골라내면 되지 않을까 생각했다

위의 코드는 실패했다 값이 음수가 되는 걸 막으려고 넣었던 값이 함수가 호출될 때 마다 초기화 된다. 그래서 차라리 값들을 배열로 빼고 정렬 시켜서 가져가면 되지 않을까해서 다시 작성해보았다

역시나 방법은 달랐지만 최솟 값을 음수로 받을 때 같은 문제가 발생했다

알고리즘을 공부해야하는 필요를 느끼고 있다.. 이래서 공부하는구나..

차라리 음수인 값들을 받지 않게 하고, 모든 태그들이 음수라 값이 없을 때 즉 null이면 실행을 안시키면 되지 않을까 싶었더니, 성공했다
